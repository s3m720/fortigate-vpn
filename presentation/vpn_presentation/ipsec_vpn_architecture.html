IPsec VPN Architecture Design

Architecture Overview

Key Components

FortiGate Firewalls: Act as VPN gateways at each end of the tunnel.
Local Networks: Internal networks at each site that need to communicate securely.
Public Network: Internet or other untrusted medium over which the encrypted tunnel is established.
Routing: Static routes or dynamic routing protocols to direct traffic through the tunnel.

IPsec Tunnel Establishment

Phase 1 (IKE): Establishes secure, authenticated channel between FortiGate devices.
  • Negotiation of encryption, hashing, DH group
  • Authentication via pre-shared key or certificates
  • Establishes IKE Security Association (SA)
Phase 2 (IPsec): Negotiates parameters for the actual data tunnel.
  • Negotiation of IPsec protocols, encryption, hashing
  • Perfect Forward Secrecy (PFS) settings
  • Establishes IPsec Security Association (SA)

Configuration Considerations

Authentication Method: Pre-shared key (PSK) or digital certificates.
Encryption & Authentication: Strong algorithms (AES256, SHA256) for both phases.
Dead Peer Detection (DPD): Detects unresponsive peers and tears down stale tunnels.
NAT Traversal: Enables IPsec traffic to pass through NAT devices.
