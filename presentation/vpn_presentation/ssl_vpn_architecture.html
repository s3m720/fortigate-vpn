SSL VPN Architecture Design

Architecture Overview

Key Components

FortiGate Firewall: Acts as the SSL VPN gateway, terminating encrypted connections.
Remote Users: Connect via FortiClient VPN client or web browser.
Internal Resources: Servers, applications, and data within the corporate network.
Authentication Server: LDAP, RADIUS, or FortiAuthenticator for user authentication.

Connection Flow

Initiation: Remote user connects to FortiGate's public IP on port 10443.
Authentication: FortiGate authenticates user against local DB or external server.
Portal/Tunnel Assignment: User assigned to SSL VPN portal (web-mode) or tunnel (full access).
Policy Enforcement: Firewall policies control access to internal resources.
Traffic Encryption: All traffic encrypted using SSL/TLS protocols.

Configuration Considerations

Interface & Port: External interface, non-standard port (10443) for security.
Server Certificate: Valid SSL certificate for secure communication.
Split Tunneling: Control whether all traffic or only corporate traffic uses VPN.
